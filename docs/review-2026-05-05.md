# コードレビュー指摘事項（2026-05-05）

ソース・ドキュメント全体の通読レビューで検出した不整合・実装不備の一覧。
後日の検討用メモ。優先度は実装影響ベースの主観値。

## 1. ドキュメント / 仕様 と 実装の不整合

### 1-1. 拡張子は受け付けるが開けない形式がある（中）

- `src/main.cpp:8-10` の `kArchiveExts[]` は `wim`, `lzma`, `lzh`, `arj` をアーカイブ判定対象として列挙。
- `docs/specification.md` および `README.md` の対応形式リストにも記載。
- 一方 `SevenZip::FormatToInGuid()` (`src/SevenZip.cpp:224`) はこれらの拡張子を扱わず、`return CLSID_Format_7z` にフォールバック。
- 結果: 7z ハンドラがマジック不一致で `S_FALSE` を返し、UI に「アーカイブを開けませんでした」が出る。

**対応方針候補**
- `sdk/7zip/Archive/IArchive.h` に `CLSID_Format_Wim (0xE6)`, `CLSID_Format_Lzh (0x06)`, `CLSID_Format_Arj (0x04)`, `CLSID_Format_Lzma (0x0A)` を追加し、`FormatToInGuid` に分岐を追加。
- 実装しないなら `kArchiveExts[]` および spec.md/README から削除。

### 1-2. 7z ヘッダ暗号化が UI 上は有効、実体は無効（高）

- `res/AileEx.rc:37` の `IDC_ENCRYPT_HDR` チェックボックスは `CompressDlg.cpp:228` で 7z 形式時に `EnableWindow(..., is7z)` され、ユーザはチェック可能。
- `OnOK` で `m_params.encryptHeaders` に値が読み出される。
- しかし `SevenZip::Compress()` (`src/SevenZip.h:41`) のシグネチャに `encryptHeaders` 引数が**ない**。`SevenZip.cpp:1095-1133` の `setProps` にも `he=on` 相当の指定なし。
- 結果: 7z で「ヘッダ暗号化」をチェックしても効かない。spec.md「7z はヘッダ暗号化オプションあり」と矛盾。

**対応方針**
- `SevenZip::Compress` に `bool encryptHeaders` を追加し、`setProps` の names/vals に `he` / `VT_BSTR("on")` を push。
- 呼出側 (`App.cpp:182`, `MainWindow.cpp:854`) を更新。

### 1-3. ドロップ経由の RAR 圧縮で詳細パラメータが反映されない（中）

- `MainWindow::OnDropFiles` (`MainWindow.cpp:402-411`) は `params.dictSize` 等は埋めるが、`rarDictSize` / `rarSolid` / `rarThreads` / `rarRecoveryPct` / `rarSplitVolume` / `rarExtra` を**埋めていない**。
- `OnAddFiles` (`MainWindow.cpp:716-731`) は正しく埋めている。
- 結果: ドラッグ&ドロップ → RAR 選択時、設定済みの RAR 詳細値が空のまま CompressDlg に渡る。`OnOK` 後の保存処理（`MainWindow.cpp:417-426`）も RAR 系を保存しないため、上書きされる可能性も。

**対応方針**
- `OnAddFiles` と同じ初期化ブロックに揃える（共通ヘルパ化が望ましい）。

### 1-4. コマンドライン経由の RAR 圧縮にキャンセル経路がない（中）

- `App::RunCompressMode` の RAR 経路 (`App.cpp:152-167`) のメッセージループは `WM_APP_DONE` と `WM_APP_PROGRESS` を捌くが、**`sink->IsCancelled()` の確認も `rarProc.Cancel()` の呼び出しもない**。
- `MainWindow::OnCompress` の `runMsgLoop` (`MainWindow.cpp:793-811`) は `cancelFn` で `rarProc.Cancel()` を呼ぶ仕組みを持っており非対称。
- 結果: コマンドライン引数で起動した圧縮の進捗ダイアログでキャンセルを押しても rar.exe が止まらない。

**対応方針**
- `App::RunCompressMode` の RAR 経路を `MainWindow::OnCompress` と同じ構造に統一、または共通関数化。
- `progDlg.SetSink(sink)` を呼んでいるので、ループ内で `sink->IsCancelled()` → `rar.Cancel()`。

### 1-5. rar.exe の ExitCode が UI に反映されない（高）

- `RarProcess::StdoutReaderThread` (`RarProcess.cpp:226-266`) はパイプ EOF 時に常に `PostMessageW(ctx->hwnd, ctx->doneMsg, 0, 0)` で **wParam=0 (= S_OK) 固定**送出。
- `GetExitCodeProcess` を呼ばないため、rar.exe が失敗終了しても UI には「完了」と表示される。
- `WinrarWaiterThread` (`RarProcess.cpp:79-94`) は ExitCode を取って `HRESULT_FROM_WIN32(ERROR_CANCELLED)` 化しており非対称。

**対応方針**
- `StdoutReaderThread` の最後で `WaitForSingleObject(hProcess, INFINITE)` → `GetExitCodeProcess` → `(ec == 0) ? S_OK : E_FAIL` を Post。
- そのために `ReaderCtx` に `HANDLE hProcess` を持たせる必要あり（`Compress` で `DuplicateHandle` して渡す）。

### 1-6. unrar.dll の更新日時がタイムゾーン分ずれる（低〜中）

- `UnrarDll::ListArchive` (`UnrarDll.cpp:96`) は `DosDateTimeToFileTime(..., &it.mtime)` をそのまま格納。
- `DosDateTimeToFileTime` の出力は **ローカル FILETIME**（DOS 時刻自体がローカル）。
- 一方 `MainWindow::PopulateList` (`MainWindow.cpp:1057`) と `InfoDlg::FormatFileTime` (`InfoDlg.cpp:11-12`) は **UTC FILETIME** 前提で `FileTimeToLocalFileTime` を適用。
- 結果: 非 UTC 環境で RAR ファイルの更新日時がタイムゾーンオフセット分ずれる（JST なら +9 時間）。

**対応方針**
- `UnrarDll.cpp:96` で `DosDateTimeToFileTime` の出力を一時 FILETIME に取り、`LocalFileTimeToFileTime(&local, &it.mtime)` で UTC 化してから格納。

### 1-7. 設定ダイアログが自動検出パスを「焼き付ける」（中）

- `SettingsDlg::OnInit` (`SettingsDlg.cpp:113-118`) は空なら自動検出値を表示。
- `SettingsDlg::OnOK` (`SettingsDlg.cpp:166-170`) は表示中の文字列をそのまま `Set*Path` で保存。
- 結果: ユーザが何も触らず OK だけ押しても、自動検出された絶対パスが固定値として INI に書かれる。後で 7-Zip がアップグレード等でパス変更されても古いパスが使われ続ける。

**対応方針**
- `OnInit` で「自動検出値かどうか」を覚えておき、編集されていなければ保存時に空文字列を書く。
- もしくは表示専用ラベル + 編集用エディットを分離（UX 変更）。

### 1-8. ドキュメント側の表現
- `docs/specification.md:60-63` の「unrar.dll では常に全展開」記述は `README.md:67` には反映されていない（README は「選択展開可能」と一般化）。
- `docs/build.md:50` 表に MSVC 18/2026 のパス（CLAUDE.md 記載値）を併記すると親切。
- `CLAUDE.md` の「対応形式」記述と `FormatToInGuid` 実装の不一致（1-1 と同件）。

## 2. 実装上の潜在バグ

### 2-1. 圧縮失敗時に既存ファイルを破壊（中）

- `SevenZip.cpp:1146-1152` の `COutFileStream::Create(outPath)` は `CREATE_ALWAYS`。
- 後続の `archive->UpdateItems` が失敗しても、開始時点で既存の同名ファイルは 0 バイトに切り詰められた状態で残る。

**対応方針**
- 一時ファイル（例: `outPath + L".part"`）に書き、成功時のみ `MoveFileEx(..., MOVEFILE_REPLACE_EXISTING)` で差し替え。

### 2-2. CExtractCallback の m_currentOut リセット漏れ（低）

- `SevenZip.cpp:709-735` の `GetStream`/`SetOperationResult` 周り。
- スキップ系（`askExtractMode != kExtract`、ディレクトリ）で `m_currentOut` がリセットされない。
- 連続して呼ばれた場合に前回の `m_currentOut` を再 Release する可能性。
- 7z.dll の現行挙動上は再現しない見込みだが脆弱。

**対応方針**
- `GetStream` 冒頭で `m_currentOut = nullptr; m_currentIsDir = false; m_currentItemIndex = -1;` をリセット。

### 2-3. パスワードのコマンドラインエスケープなし（低）

- `RarProcess.cpp:128-132` で `cmd += L" -hp" + pw` / `L" -p" + pw`。
- パスワードに空白・`"`・`^` 等が含まれると CreateProcessW のパース崩れ。
- パスワードファイル `-p@filename` 経由などのほうが安全。

### 2-4. CreateProcessW の bInheritHandles=TRUE（低）

- `RarProcess.cpp:192` (Console モード) は `bInheritHandles=TRUE`。
- 親プロセスの全 inheritable ハンドルが継承される。
- `STARTUPINFOEXW` + `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` で stdout のハンドルだけに絞るのが望ましい。

### 2-5. MainWindow::OnProgress / OnDone は事実上デッドコード（低・整理）

- `MainWindow::OnExtract` の入れ子 MSG ループ (`MainWindow.cpp:646-663`) と `OnCompress::runMsgLoop` (`MainWindow.cpp:793-811`) が `WM_APP_PROGRESS` を `continue` で吸収するため、`HandleMsg` 内の `WM_APP_PROGRESS` → `OnProgress` 経路 (`MainWindow.cpp:231-233`) には到達しない。
- 防御的コードとして残すのは可。残すなら `// fallback` コメントを書くと意図が伝わる。

### 2-6. アーカイブ Open 時のパスワード未対応（中）

- 暗号化ヘッダ付きの 7z などを開くと `COpenCallback::CryptoGetTextPassword` (`SevenZip.cpp:98`) が空文字列の BSTR を返し、結果 Open が失敗。
- パスワード入力 UI が存在しない。

**対応方針**
- `OpenArchive` が `E_ACCESSDENIED` 系の HRESULT または特定の失敗パターンを返したらパスワード入力ダイアログを出してリトライ。
- API 側にも `password` 引数があるので（`SevenZip.h:30`）、UI 取得 → 渡す経路を追加すれば対応可能。

### 2-7. Settings::Save の事前条件（低）

- `Settings.cpp:54` の `Save` は `m_iniPath` が `Load` 経由で設定済みであることに依存（`BuildIniPath` を呼ばない）。
- 現状 `App::Init` 内で必ず `Load` → `Save` の順に呼ばれているが、初期化順を呼び出し規約に依存。
- `Save()` 冒頭で `if (!m_iniPath[0]) BuildIniPath();` の防御を入れてもよい（`Save` の `const` 性は維持）。

### 2-8. CompressDlg の表示揺れ（最低）

- `src/CompressDlg.cpp:55` `kMethodsRar` の "Normal" 行のラベルに 1 行だけ二重スペースがある。整列の見栄え程度。
